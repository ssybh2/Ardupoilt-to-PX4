PX4 v1.17.0 L1 控制器最简操作指导

工程目录：

  /home/aim/px4_ws/PX4-Autopilot-v1.17.0

本指导只用于 Gazebo Classic UI 仿真，不要用于真实飞机。


一、先清理残留仿真

如果上一次仿真异常退出，先执行：

cd /home/aim/px4_ws/PX4-Autopilot-v1.17.0
build/px4_sitl_default/bin/px4-shutdown 2>/dev/null || true
pkill -x gzclient 2>/dev/null || true
pkill -x gzserver 2>/dev/null || true
pkill -x gazebo 2>/dev/null || true
pkill -f build/px4_sitl_default/bin/px4 2>/dev/null || true
pkill -f sitl_run.sh 2>/dev/null || true

确认没有残留：

ps -eo pid,comm,args | awk '$2=="px4" || $2=="gzserver" || $2=="gzclient" || $2=="gazebo" || $2=="sitl_run.sh" {print}'

如果没有输出，说明已经清理干净。


二、方式 A：需要键盘操控时使用

键盘 w/s/0/r/q 需要真实可交互终端，所以必须让 PX4 进入 pxh>。

打开一个普通终端，执行：

cd /home/aim/px4_ws/PX4-Autopilot-v1.17.0
source /home/aim/px4_ws/px4-venv/bin/activate
unset HEADLESS
unset NO_PXH
make px4_sitl gazebo-classic_iris

正常现象：

1. Gazebo Classic UI 打开。
2. Iris 四旋翼出现。
3. 终端出现 pxh>。

如果出现 pxh> 后马上 Exiting NOW，或 pxh> 高速刷新，说明当前终端不能做键盘操控。请换一个真实桌面终端再试，或使用下方“方式 B”。


三、启动 L1 并起飞

看到 pxh> 后输入：

mc_rate_control stop
l1_adaptive_control start
commander arm

如果 arm 被 EKF 或 preflight 拒绝，等 3-5 秒后再输入一次：

commander arm

预期现象：

1. 飞机解锁。
2. 自动起飞到约 1 米。
3. 起飞完成后进入悬停。

检查状态：

l1_adaptive_control status
listener vehicle_local_position 1

成功时重点看：

armed=1
failsafe=0
valid=1
pipeline: traj=1 geo=1 l1=1 publish=1
vehicle_local_position 的 z 接近 -1.0


四、随时切换轨迹

查看当前轨迹：

l1_adaptive_control trajectory status

切到圆轨迹：

l1_adaptive_control trajectory circle

切回悬停：

l1_adaptive_control trajectory hover

建议先等飞机起飞并稳定悬停后，再切换圆轨迹。


五、随时启动键盘操控

先确认飞机已经起飞或正在悬停，然后在 pxh> 输入：

l1_adaptive_control rc_control enable
l1_keyboard_throttle start

启动后当前终端会进入键盘读取状态，不会继续显示 pxh>。
此时直接按键，不需要回车。

按键：

w  目标高度上升
s  目标高度下降
x 或空格  停止继续上升/下降，保持当前目标高度
0  固定失能 Motor 1
r  恢复 Motor 1
q  退出键盘工具，并把 throttle 清零

飞圆形轨迹时也可以使用同一套高度控制：

1. 先执行 l1_adaptive_control trajectory circle。
2. 再执行 l1_adaptive_control rc_control enable 和 l1_keyboard_throttle start。
3. 按 w 上升，按 s 下降，按 x 或空格保持当前高度。
4. 按 q 退出键盘工具后，可以继续切换 trajectory hover/circle。


六、结束键盘操控

在键盘工具运行时按：

q

回到 pxh> 后输入：

l1_adaptive_control rc_control disable

查看是否关闭：

l1_adaptive_control rc_control status

看到 RC height control: disabled 即可。


七、结束仿真

如果正在键盘工具里，先按：

q

然后在 pxh> 输入：

shutdown

也可以按 Ctrl+C 结束仿真。当前工作区已经加了退出兜底：第一次 Ctrl+C 会正常请求 PX4 退出；如果 5 秒内仍卡在 PX4 Exiting，会自动强制结束；第二次 Ctrl+C 会立即强制结束。

如果 pxh> 已经不可用，可以打开另一个终端执行：

cd /home/aim/px4_ws/PX4-Autopilot-v1.17.0
build/px4_sitl_default/bin/px4-shutdown

如果仍有残留，再执行第一节“清理残留仿真”。


八、方式 B：pxh 不可用时的备用起飞和轨迹

如果你的终端无法稳定使用 pxh>，可以用后台模式启动并用 px4-* client 命令控制。这个方式能起飞和切换轨迹，但不能使用键盘 w/s/0/r/q。

启动：

cd /home/aim/px4_ws/PX4-Autopilot-v1.17.0
source /home/aim/px4_ws/px4-venv/bin/activate
unset HEADLESS
export NO_PXH=1
make px4_sitl gazebo-classic_iris > /tmp/px4_l1_ui.log 2>&1 &

等待启动：

sleep 18

启动 L1 并起飞：

build/px4_sitl_default/bin/px4-mc_rate_control stop
build/px4_sitl_default/bin/px4-l1_adaptive_control start
sleep 8
build/px4_sitl_default/bin/px4-commander arm

检查：

build/px4_sitl_default/bin/px4-l1_adaptive_control status
build/px4_sitl_default/bin/px4-listener vehicle_local_position 1

轨迹：

build/px4_sitl_default/bin/px4-l1_adaptive_control trajectory status
build/px4_sitl_default/bin/px4-l1_adaptive_control trajectory circle
build/px4_sitl_default/bin/px4-l1_adaptive_control trajectory hover

结束：

build/px4_sitl_default/bin/px4-shutdown


九、常见问题

1. 输入 make 后出现 pxh> Exiting NOW。

当前终端不能作为 PX4 交互 shell。需要键盘操控时，换一个真实桌面终端重新执行方式 A；只需要起飞和轨迹时，用方式 B。

2. 输入 commander arm 后不飞。

确认先执行了：

mc_rate_control stop
l1_adaptive_control start

然后等 8 秒再 arm。

3. 按 w/s 没反应。

确认已经执行：

l1_adaptive_control rc_control enable
l1_keyboard_throttle start

并且当前终端停在 l1_keyboard_throttle 工具里。键盘工具运行时不会显示 pxh>。

4. 按 0 后飞机不稳。

0 会真实注入 Motor 1 off。普通四旋翼单电机完全失能后不保证还能稳定悬停。

5. MAVLink only on localhost 是错误吗？

不是。这只是提示 MAVLink 只监听本机。

6. poll timeout 是错误吗？

不一定。Gazebo UI 卡顿时可能出现 poll timeout。当前工作区已经把这类日志限频，避免它持续刷屏影响 pxh> 输入。只要飞机能起飞、status 正常、位置 z 接近 -1.0，就先不要把它当作 L1 控制器失败。
